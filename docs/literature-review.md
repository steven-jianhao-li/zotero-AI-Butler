# 📚 文献综述生成

AI管家支持对多篇论文进行综合分析，自动生成文献综述报告。

## 功能介绍

**文献综述生成**功能可以让您：

- 选择分类下的多篇论文
- 自动提取所有 PDF 内容
- 调用大模型生成综合文献综述
- 创建独立的报告条目并关联所有原始 PDF

## 使用方法

### 步骤 1：打开文献综述配置

在 Zotero 中，右键点击任意**分类（Collection）**，选择 **"AI管家文献综述"**。

![右键分类打开文献综述](./images/literature-review-context-menu.png)

### 步骤 2：配置综述参数

在弹出的配置页面中：

1. **综述名称**：为您的综述报告命名（默认为当前日期）
2. **自定义提示词**：可以修改默认提示词以满足特定需求
3. **选择论文**：勾选需要纳入综述的论文
   - 论文列表显示分类下所有有 PDF 附件的文献
   - 如果论文有多个 PDF，可展开查看并单独选择

### 步骤 3：生成综述

点击 **"🚀 生成综述"** 按钮，AI管家将：

1. 创建新的报告条目
2. 将选中的 PDF 作为链接附件添加到报告
3. 读取所有 PDF 内容
4. 调用大模型生成综合文献综述
5. 保存综述笔记到报告条目下

## 生成的内容

成功生成后，您的分类下会新增一个 **"报告"** 类型的条目，包含：

- **PDF 链接附件**：所有选中的原始 PDF 的链接
  - 附件命名格式：`[论文标题前30字] 原附件名`
- **AI 笔记**：自动生成的综合文献综述

## 默认提示词

默认的文献综述提示词会指导 AI 生成以下内容：

1. **研究主题概述**：论文共同关注的研究领域和核心问题
2. **各论文主要贡献**：逐一总结每篇论文的核心观点、方法和发现
3. **研究方法对比**：分析各论文研究方法的异同
4. **主要发现汇总**：综合各论文的主要结论和发现
5. **研究趋势与展望**：分析该领域的发展趋势和未来研究方向

## 技术说明

- **多模态支持**：使用 Gemini API 的 `inline_data` 方式发送多个 PDF
- **大文件处理**：使用分块 Base64 编码，支持大型 PDF 文件
- **自动命名**：PDF 链接附件自动添加论文标题前缀，便于识别

## 注意事项

1. **API 额度**：综述生成会一次性发送多个完整 PDF，请注意 API 额度消耗
2. **文件大小**：单次请求的总文件大小受 API 限制，建议单次不超过 10 篇论文
3. **支持模型**：目前仅支持 Gemini 多模态模型进行多文件综述
